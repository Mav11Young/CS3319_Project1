\begin{abstract}
This report investigates dimensionality reduction for image classification
on the Animals with Attributes~2 (AwA2) benchmark.
We start from 2048-dimensional ResNet-101 features and evaluate a linear
Support Vector Machine (SVM) baseline, then apply three complementary
dimensionality-reduction strategies---ANOVA-based feature selection
(\textsc{SelectKBest}), Principal Component Analysis (PCA), and
sparse coding with OMP---across a wide range of target dimensions.
The baseline linear SVM achieves \textbf{92.52\%} test accuracy on the
full 2048-d representation.
After dimensionality reduction, none of the three methods surpasses the
baseline; the best reduced representation (SelectKBest at 768-d) reaches
\textbf{92.16\%}, showing only a small performance drop with substantial
compression.
Systematic experiments reveal that PCA provides the best trade-off between
accuracy and compression, while sparse coding is consistently the weakest
method under our fast OMP configuration.
We identify the optimal method and dimensionality for this dataset and
provide detailed analyses of performance variance across the full
dimensionality spectrum.
\end{abstract}

\section{Experimental Setting}
\label{sec:setting}

\paragraph{Dataset.}
We use the Animals with Attributes~2 (AwA2) benchmark~\cite{xian2019zero,
lampert2009learning}, a widely used dataset for zero-shot and
fine-grained recognition.
AwA2 contains \textbf{37,322 images} spanning \textbf{50 animal categories}
with rich semantic variation (from common mammals to marine species).
For each image, the dataset provides pre-extracted \textbf{2048-dimensional
ResNet-101}~\cite{he2016deep} feature vectors obtained from the penultimate
layer of a network pre-trained on ImageNet.
These deep features serve as the sole input representation throughout all
experiments; raw pixels are never used.

\paragraph{Train/test split.}
Following the project specification, images within each of the 50 classes
are split independently into \textbf{60\% training} and
\textbf{40\% testing}, using a fixed random seed (42) for reproducibility.
This per-class stratified split yields \textbf{22,373 training samples} and
\textbf{14,949 test samples}.
Class-level split sizes range from 40 (minimum) to 787 (maximum) test
samples, reflecting the natural class-size imbalance in AwA2.
All dimensionality-reduction methods and classifier hyperparameter searches
operate exclusively on the training split; the test set is never used during
any fitting step.

\paragraph{Preprocessing.}
Before applying any dimensionality-reduction method we fit a
\texttt{StandardScaler} on the training features and apply the same
transformation to the test features.
Zero-centering and unit-variance normalisation ensures that features with
large magnitudes do not dominate the ANOVA F-statistics, PCA covariance
computation, or sparse-coding objective.

\paragraph{Classifier.}
Across all experiments (baseline and dimensionality-reduced variants) we
use a \textbf{linear SVM} implemented by \texttt{LinearSVC}
\cite{fan2008liblinear,cortes1995support} from scikit-learn
\cite{scikit-learn}.
For Q3/Q4, to keep runtime manageable across many method-dimension settings,
we reuse the best value from Q1/Q2 ($C=0.01$) for all LinearSVC fits instead
of re-running cross-validation per setting.
The solver uses the primal formulation (\texttt{dual=False}) with a
maximum of 5000 iterations.

\section{Dimensionality Reduction Methods}
\label{sec:methods}

We evaluate three fundamentally different strategies for reducing the
2048-d feature vectors to a lower-dimensional representation.

\subsection{Method~1: Feature Selection via \textsc{SelectKBest}}
\label{sec:sel}

\paragraph{Motivation.}
Feature selection retains a subset of the \emph{original} dimensions and
discards the rest.
It is interpretable (selected features correspond to concrete ResNet-101
channels), computationally cheap, and preserves feature semantics.

\paragraph{Method.}
We apply the \textbf{ANOVA F-test} (one-way analysis of variance) to score
each feature dimension by its ability to discriminate among the 50 classes
\cite{fisher1992statistical,guyon2003introduction}.
For a feature $j$, the F-statistic is:
\begin{equation}
    F_j = \frac{\text{between-class variance of } x_j}
               {\text{within-class variance of } x_j}.
    \label{eq:ftest}
\end{equation}
The top-$k$ features by $F_j$ are retained; all others are zeroed out.
Because the scoring is invariant to the target dimensionality $k$, we
compute the F-statistics \emph{once} and subset to the desired $k$ by
index selection, making the procedure highly efficient.

\paragraph{Target dimensionalities.}
$k \in \{32, 64, 128, 256, 512, 768, 1024, 1280, 1536, 2048\}$.
The case $k=2048$ recovers the full (scaled) feature set and serves as a
within-method sanity check.

\subsection{Method~2: Feature Projection via PCA}
\label{sec:pca}

\paragraph{Motivation.}
Principal Component Analysis \cite{pearson1901pca,hotelling1933analysis}
constructs a new orthonormal basis in which the directions of maximum
variance are ranked first.
Unlike feature selection, PCA produces \emph{linear combinations} of all
original features, capturing global structure and eliminating correlated
dimensions simultaneously.

\paragraph{Method.}
Let $\mathbf{X}_{\text{train}} \in \mathbb{R}^{N \times D}$ be the
(standardised) training features.
PCA computes the eigendecomposition of the sample covariance matrix:
\begin{equation}
    \mathbf{C} = \frac{1}{N-1}\mathbf{X}_{\text{train}}^\top
                 \mathbf{X}_{\text{train}} \in \mathbb{R}^{D \times D}.
\end{equation}
For a target dimension $k$, the training and test features are projected
onto the top-$k$ eigenvectors $\mathbf{W}_k \in \mathbb{R}^{D \times k}$:
\begin{equation}
    \tilde{\mathbf{x}} = \mathbf{W}_k^\top \mathbf{x}.
    \label{eq:pca}
\end{equation}
We fit PCA with the largest required $k$ and slice the score matrix for
smaller values of $k$, which is both exact and computationally efficient.

\paragraph{Target dimensionalities.}
$k \in \{32, 64, 128, 256, 512, 768, 1024, 1536\}$.

\subsection{Method~3: Feature Learning via Sparse Coding (OMP)}
\label{sec:sc}

\paragraph{Motivation.}
Both feature selection and PCA impose \emph{linearity} on the
representation.
sparse coding learns compact representations by expressing each
input feature vector as a sparse combination of learned dictionary atoms.
The resulting sparse code is the learned low-dimensional feature.

\paragraph{Method.}
For each target dimensionality $k$, we first initialise a dictionary
$\mathbf{D}\in\mathbb{R}^{k\times 2048}$ using either PCA components
(default) or random training samples, then infer a sparse code
$\mathbf{z}\in\mathbb{R}^{k}$ for each sample by solving:
\begin{equation}
    \min_{\mathbf{z}} \frac{1}{2}\left\|\mathbf{x}-\mathbf{D}^{\top}\mathbf{z}\right\|_2^2
    + \lambda \|\mathbf{z}\|_1,
\end{equation}
and solve the coding step with Orthogonal Matching Pursuit (OMP) under a
fixed sparsity level (number of non-zero coefficients).
The inferred sparse codes are used as low-dimensional learned features for
LinearSVC classification.

\paragraph{Training.}
Dictionary initialisation and sparse-code inference are both built on the
standardised training split. For each $k$, we record reconstruction MSE and
code density (non-zero ratio) as diagnostics of representation quality and
sparsity.

\paragraph{Target dimensionalities.}
$k \in \{32, 64, 128, 256, 512, 1024\}$.

    \label{fig:acc_vs_dim}
\end{figure*}

Figure~\ref{fig:acc_vs_dim} presents the main result of Question~3:
test accuracy as a function of the number of retained dimensions across
all three methods.
Several consistent patterns emerge.

\paragraph{Low-dimensional regime ($k \le 128$).}
All methods degrade significantly below 128 dimensions.
Sparse coding is noticeably weaker than PCA and SelectKBest even in this
regime, indicating that reconstruction-driven sparse codes do not align
well with class-discriminative directions here.
PCA and SelectKBest converge more quickly as $k$ grows, because their
linear projections need fewer dimensions to preserve the dominant
class-discriminative signal already present in the ResNet-101 space.

\paragraph{Mid-dimensional regime ($128 \le k \le 512$).}
PCA and SelectKBest recover most of the baseline accuracy.
PCA reaches near-baseline performance earliest, consistent with
the observation from Figure~\ref{fig:pca_variance} that the first few
hundred PCs already explain the large majority of total variance.

\paragraph{High-dimensional regime ($k \ge 512$).}
No method exceeds the baseline.
SelectKBest and PCA plateau slightly below the baseline, while sparse coding
remains clearly lower. This indicates that, on this dataset, keeping the
full 2048-d representation is still best for peak accuracy.

\section{Optimal Method and Dimensionality}
\label{sec:optimal}

\subsection{Finding the Global Optimum}

\begin{figure*}[t]
    \centering
    \includegraphics[width=\linewidth]{figures/q4_optimal_perclass.pdf}
    \caption{%
        \emph{Left}: test-accuracy change (in percentage points, pp) relative
        to the 2048-d baseline for each method at its best dimensionality.
        \emph{Right}: per-class baseline accuracy distribution ranked from
        hardest (red) to easiest (green), showing which categories are most
        susceptible to improvement by dimensionality reduction.}
    \label{fig:optimal}
\end{figure*}

Table~\ref{tab:summary} and Figure~\ref{fig:optimal} together address
Question~4.
The globally optimal configuration is:
\begin{itemize}
    \item \textbf{Method}: SelectKBest (ANOVA F-test)
    \item \textbf{Dimension}: $k^* = 768$
    \item \textbf{Test accuracy}: 92.16\%
    \item \textbf{Accuracy gain over baseline}: $-0.36$ pp (slight drop)
\end{itemize}

\subsection{Why This Configuration Works}

\paragraph{PCA as a strong projection baseline.}
PCA exploits the correlation structure of ResNet-101 features.
Because deep network activations across similar images are highly
correlated, the top principal components capture the dominant
discriminative variance while discarding noise.
For AwA2, where each of the 50 classes has a tight visual prototype
(e.g.\ stripes for \textit{zebra}, tusks for \textit{elephant}), a
linear projection is sufficient to separate the most informative subspace.

\paragraph{Feature selection is competitive and interpretable.}
ANOVA-selected channels correspond to specific ResNet-101 feature-map
positions that are maximally activated by class-discriminative textures,
shapes, or colours.
Their strong performance confirms that the redundancy in the 2048-d space
is distributed: the top-ranked channels already encode most of the
classification signal.

\paragraph{Sparse coding is the weakest method in this experiment.}
Sparse coding underperforms both SelectKBest and PCA at all tested
dimensions (best 87.38\% at 64-d).
Two practical reasons explain this gap: (1) the objective is reconstruction
rather than classification, so sparse codes are not explicitly optimised for
class separation; (2) with fixed OMP sparsity and fixed $C$, code density
drops quickly as dimensionality grows, reducing effective discriminative
capacity for LinearSVC.

\paragraph{Optimal dimensionality regime.}
The consistent empirical finding across all three methods is that the
\emph{sweet spot} lies around 256--1024
dimensions.
Below this range, accuracy degrades because the representation is too
compressed to retain all 50-class discriminative boundaries.
Above it, accuracy plateaus as the marginal information added per
dimension becomes negligible.
This suggests that the intrinsic \emph{effective dimensionality} of the
AwA2 ResNet-101 feature manifold is well below 2048, consistent with the
PCA variance analysis showing $>$95\% variance explained by the top few
hundred components (Figure~\ref{fig:pca_variance}).

\paragraph{Compression ratio.}
At the best reduced dimensionality $k^* = \textbf{768}$, the features are
compressed by a factor of $\lfloor 2048 / k^* \rfloor \approx
\textbf{2.67}{\times}$ with moderate accuracy loss,
demonstrating highly practical redundancy reduction.

\section{Analysis and Observations}
\label{sec:analysis}

\paragraph{Comparison of reduction paradigms.}
Feature selection is the most efficient approach at inference time, as
it simply indices into the original feature vector; no matrix
multiplication is required.
PCA introduces a matrix-vector product but achieves better accuracy
at low dimensionality because orthogonal projection distributes
information across all components.
sparse coding introduces additional dictionary initialisation and sparse
inference cost, but provides learned representations beyond pure
selection/projection and explicit sparsity control.

\paragraph{Effect of StandardScaler.}
Pre-scaling the ResNet-101 features before all reduction methods is
important.
Without scaling, high-magnitude feature channels could dominate the
ANOVA F-statistics (biasing SelectKBest), inflate certain principal
components (biasing PCA), or skew sparse-code inference due to
unbalanced magnitudes.

\paragraph{Hyperparameter sensitivity.}
In the final Q3/Q4 run we used a fixed $C=0.01$ (the Q1/Q2 optimum) for
all method-dimension settings to reduce runtime and keep comparisons
consistent.
This may slightly under-estimate the best possible sparse-coding
performance at some dimensions, but does not affect the qualitative
ranking between methods in our experiments.

\paragraph{Class-level effects.}
The per-class accuracy distribution at the optimal setting
(Figure~\ref{fig:optimal}, right) shows that the hard classes
(\textit{mouse}, \textit{blue whale}, \textit{rat}) remain difficult
regardless of the reduction method.
Their low accuracy is intrinsic to limited sample size and visual
ambiguity, not to the dimensionality.
Conversely, easy classes (\textit{tiger}, \textit{giraffe}) remain at
near-100\% under all settings, confirming that dimensionality reduction
does not harm well-separated classes.

\paragraph{Practical recommendation.}
For practitioners deploying a classifier on AwA2-style data, PCA with
$k \approx 256$ dimensions provides the best balance of
accuracy, compression, and computational cost.
SelectKBest at the same $k$ is preferable when model interpretability or
feature-attribution is required.
Sparse coding should be considered when sparse, learned features are
preferred and explicit control over code sparsity is desired.

\section{Conclusion}
\label{sec:conclusion}

We have systematically studied dimensionality reduction for image
classification on AwA2 using three representative methods: ANOVA-based
feature selection, PCA, and sparse coding with OMP.
The baseline linear SVM on 2048-d ResNet-101 features achieves
\textbf{92.52\%} test accuracy.
None of the tested dimensionality-reduction methods exceeds this baseline;
the best reduced result is SelectKBest achieving 92.16\% at
$k^* = 768$ dimensions, representing a $2.67\times$ compression ratio.

The key findings are:
(1) The effective intrinsic dimensionality of AwA2's ResNet-101 feature
space is substantially below 2048---approximately 256--768
dimensions suffice to retain near-full discriminative power;
(2) PCA outperforms SelectKBest at the same low dimensionality because
projection distributes information more efficiently than selection;
(3) sparse coding is consistently the weakest method in this setup,
mainly because reconstruction-oriented sparse codes are less aligned with
linear classification boundaries;
(4) the hardest classes remain hard regardless of dimensionality,
indicating that the bottleneck is sample size and inter-class similarity,
not feature redundancy.

Future work could explore supervised dimensionality reduction (e.g.\
Linear Discriminant Analysis), non-linear projection methods (e.g.\
t-SNE, UMAP), or combined selection+projection pipelines to further
push the accuracy-compression frontier.
